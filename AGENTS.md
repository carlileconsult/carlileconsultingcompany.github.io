# AGENTS.md

## Project purpose
This repository contains the public-facing website for Carlile Consulting Company.

The site should present the company as:
Practical IT, software, and AI consulting for businesses.

## Business intent
The website should:
- look professional and credible
- explain services clearly
- encourage prospective clients to reach out
- support future lead qualification with an AI intake assistant

## Audience
Small and midsize businesses looking for practical help with:
- IT consulting
- software and systems consulting
- workflow/process automation
- AI adoption and implementation

## Brand voice
Use a practical, trustworthy, business-friendly tone.
Avoid hype, buzzwords, and exaggerated AI claims.
Prefer clarity over cleverness.

## Technical constraints
- Keep the site static and compatible with GitHub Pages
- Prefer plain HTML, CSS, and lightweight JavaScript
- Do not add heavy frameworks unless explicitly requested
- Reuse and improve the current structure where possible
- Keep changes simple, maintainable, and easy to review

## Content guidance
Emphasize:
- practical solutions
- business outcomes
- clear process
- credibility and experience

Avoid:
- vague marketing language
- overpromising results
- generic chatbot copy

## Preferred site structure
- Home
- Services
- How It Works
- About
- Contact / Start a Project

## AI assistant guidance
Do not implement a real AI assistant unless explicitly requested.
It is acceptable to create a placeholder or CTA for a future AI intake assistant if asked.

## Working style
For larger tasks:
1. inspect the repo
2. propose a plan
3. make phased changes
4. summarize what changed

When editing copy:
- keep it concise
- business-focused
- easy to skim
- professional but approachable
